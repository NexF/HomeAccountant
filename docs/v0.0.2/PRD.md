## 1. 版本概述

### 1.1 版本目标

v0.0.2 在 MVP 基础上新增三大财务管理模块，将"记账工具"升级为"家庭财务管理平台"：

| 模块 | 核心价值 |
|------|---------|
| 固定资产折旧管理 | 反映资产的真实价值变化，避免"买了手机 8000 元永远显示 8000" |
| 贷款管理 & 还款计划 | 清晰掌握每笔贷款的还款进度、利息成本，合理规划现金流 |
| 预算管理 & 提醒 | 控制月度费用支出，超支预警，培养理性消费习惯 |

### 1.2 与 v0.0.1 的关系

- **数据模型层**：v0.0.1 已预定义 `fixed_assets`、`loans`、`budgets` 三张表，v0.0.2 在此基础上进行小幅扩展
- **科目体系**：复用 v0.0.1 的五大类科目体系，新增折旧/利息/预算相关自动分录
- **记账核心**：复用 v0.0.1 的"购买资产"、"借入/贷款"、"还款"记账类型，与新模块联动
- **报表**：资产负债表、损益表已包含折旧费用和利息支出科目，数据自动汇入

### 1.3 不包含的内容（留待后续版本）

- 家庭账本 & 多人协作
- CSV 账单导入 & 流水匹配对账
- 高级模式（手动分录）
- 数据导出（CSV/Excel/PDF）

## 2. 功能需求：固定资产折旧管理

### 2.1 功能概述

对家庭持有的固定资产（房产、汽车、电子设备、家电等）进行折旧管理，在资产负债表中反映资产的**账面净值**（原值 - 累计折旧），而非一成不变的购入价格。

### 2.2 核心概念

| 概念 | 说明 |
|------|------|
| 原值 (Original Cost) | 资产购入时的金额 |
| 残值率 (Residual Rate) | 资产报废时预计残留价值的比例，默认 5% |
| 使用寿命 (Useful Life) | 预计可使用的月数 |
| 折旧粒度 (Depreciation Granularity) | `monthly`（按月折旧）或 `daily`（按日折旧），默认 `monthly` |
| 折旧方法 | 直线法（平均折旧）；暂不支持加速折旧 |
| 累计折旧 (Accumulated Depreciation) | 自购入以来已计提的折旧总额 |
| 账面净值 (Net Book Value) | 原值 - 累计折旧 |
| 每期折旧额 | 按月：(原值 × (1 - 残值率)) / 使用寿命月数；按日：(原值 × (1 - 残值率)) / 使用寿命天数 |

### 2.3 折旧计算公式

**直线法（Straight-Line Method）：**

**按月折旧（默认）：**
```
月折旧额 = (原值 × (1 - 残值率)) / 使用寿命月数
```

示例：iPhone 16，原值 8,000 元，残值率 5%，使用寿命 36 个月
```
月折旧额 = 8,000 × (1 - 5%) / 36 = 8,000 × 0.95 / 36 ≈ 211.11 元/月
```

**按日折旧：**
```
使用寿命天数 = 使用寿命月数 × 30（或根据实际自然日计算）
日折旧额 = (原值 × (1 - 残值率)) / 使用寿命天数
```

示例：iPhone 16，原值 8,000 元，残值率 5%，使用寿命 36 个月（= 1,080 天）
```
日折旧额 = 8,000 × (1 - 5%) / 1080 = 8,000 × 0.95 / 1080 ≈ 7.04 元/天
```

> **使用寿命天数计算方式：** 默认采用 `使用寿命月数 × 30` 的简化方式。如需精确计算，可根据购入日期推算到期日期，取实际自然日数（考虑大小月和闰年）。用户在新建资产时可选择计算方式。

### 2.4 自动折旧分录

系统按照资产设定的折旧粒度自动生成折旧分录（定时任务或手动触发）：

**按月折旧示例：**
```
借：折旧费用 (5014)    211.11
  贷：累计折旧 (1502)    211.11
```

**按日折旧示例：**
```
借：折旧费用 (5014)    7.04
  贷：累计折旧 (1502)    7.04
```

**触发时机：**
- **按月自动模式**：每月 1 日凌晨，定时任务自动为所有 `depreciation_granularity=monthly` 的活跃固定资产生成上月折旧分录
- **按日自动模式**：每日凌晨，定时任务自动为所有 `depreciation_granularity=daily` 的活跃固定资产生成前一日折旧分录
- **手动模式**：用户可在固定资产详情页手动触发"计提折旧"（按月资产触发一个月，按日资产触发一天）

**折旧终止条件：**
- 累计折旧达到 `原值 × (1 - 残值率)` 时停止折旧
- 资产状态变为 `disposed`（已处置）时停止

### 2.5 与记账模块联动

**购买资产时在表单内直接设置折旧：**

用户选择"购买资产"类型记账时，根据所选资产科目**智能展示**折旧设置区域，无需弹窗跳转：

1. 基础字段（必填）：
   - 金额（资产原值）
   - 资产科目（从 `asset` 类型科目中选择，如固定资产、短期投资等）
   - 支付账户
   - 贷款（可选，用于贷款购买场景）

2. 折旧设置（根据资产科目自动控制）：
   - **当资产科目选择「固定资产（1501）」时**：自动展开折旧设置区域，且为**必填**
   - **当资产科目选择其他科目**（如短期投资、银行存款等）：折旧设置区域**隐藏**，不需要填写
   - 折旧设置字段：
     - 资产名称（如"iPhone 16 Pro"、"Model 3"），必填
     - 使用寿命（月），默认 36 个月
     - 残值率，默认 5%
     - 折旧方式：直线法（默认）/ 不折旧
     - 折旧粒度：按月（默认）/ 按日

3. 保存时一次性完成：
   - 生成分录：`借：资产科目 / 贷：银行存款/信用卡`
   - 若资产科目为固定资产，自动创建 `fixed_assets` 记录并关联到对应科目，开始按设定的粒度自动折旧
   - 若资产科目非固定资产，则只生成分录，不创建固定资产记录

**设计原则：** 一次操作完成记账 + 资产登记，避免弹窗中断用户操作流程。折旧设置与资产科目联动——选择固定资产科目时强制要求填写折旧信息，其他科目（如短期投资）则无需折旧。

**处置资产：**

用户可在固定资产详情页"处置资产"：
1. 输入处置收入（如卖二手手机得 2,000 元）
2. 系统自动生成分录：
```
借：银行存款          2,000    （收到的钱）
借：累计折旧          5,700    （已折旧部分转出）
借：资产处置损失        300    （如果亏了，差额计入费用）
  贷：固定资产         8,000    （原值冲销）
```
或者如果赚了：
```
借：银行存款          3,000
借：累计折旧          5,700
  贷：固定资产         8,000
  贷：资产处置收益       700    （差额计入收入）
```
3. 资产状态变为 `disposed`

## 3. 功能需求：贷款管理 & 还款计划

### 3.1 功能概述

对家庭持有的贷款（房贷、车贷、消费贷等）进行全生命周期管理，包括：贷款信息录入、还款计划生成、每期还款金额的本金/利息自动拆分、还款进度追踪。

### 3.2 核心概念

| 概念 | 说明 |
|------|------|
| 贷款本金 (Principal) | 贷款总额 |
| 剩余本金 (Remaining Principal) | 尚未偿还的本金 |
| 年利率 (Annual Rate) | 贷款年化利率 |
| 月利率 | 年利率 / 12 |
| 贷款期数 (Total Months) | 贷款总月数 |
| 等额本息 (Equal Installment) | 每月还款金额固定，前期利息多、后期本金多 |
| 等额本金 (Equal Principal) | 每月偿还等额本金 + 递减利息，月供逐月递减 |

### 3.3 还款计算公式

**等额本息：**
```
月供 = P × r × (1+r)^n / ((1+r)^n - 1)

其中：P = 贷款本金，r = 月利率，n = 贷款总月数

每期利息 = 剩余本金 × 月利率
每期本金 = 月供 - 每期利息
```

**等额本金：**
```
每期本金 = P / n（固定）
每期利息 = 剩余本金 × 月利率
每期月供 = 每期本金 + 每期利息（逐月递减）
```

### 3.4 还款计划表

系统根据贷款信息自动生成完整的还款计划表：

### 3.5 与记账模块联动

**方式一：通过"记一笔"→"借入"（支持贷款联动）**

用户在"记一笔"中选择"借入"类型时：
1. 填写金额、负债科目、收款账户（必填）
2. 可选开启「贷款设置」面板，填写：贷款名称、年利率、还款期数、还款方式、首次还款日期
3. 开启贷款设置后，页面实时显示还款预估（月供、利息总额、还款总额）
4. 提交后系统自动：
   - 生成借款入账分录（借资产，贷负债）
   - 若开启了贷款设置，同时创建贷款记录（`loans` 表）
5. 不开启贷款设置则为简单借入记录，适用于亲友借款等无利息场景

**方式二：通过贷款管理模块直接新建**

用户在贷款管理中新建贷款时：
1. 填写贷款基本信息（名称、负债科目、金额、利率、期数、还款方式、起始日期）
2. 可选择「放款到账户」（资产账户，如银行存款）
3. 若选择了放款账户，系统自动生成借款入账分录：
```
借：银行存款（放款账户）  2,000,000
  贷：长期借款（负债科目）  2,000,000
```
4. 系统自动计算月供金额，创建 `loans` 记录
5. 系统自动生成完整还款计划表

> **两种方式对比：** "记一笔→借入"更适合快速记录，可选是否关联贷款；"新建贷款"更专业，必填贷款详情。两者最终效果一致。

**还款时自动更新贷款记录：**

用户在贷款详情页点击"本期还款"：
1. 系统根据还款计划**自动计算**本期应还的本金和利息
2. 用户选择还款账户（资产账户）和利息科目
3. 生成分录：
```
借：长期借款（本金）  2,281
借：利息支出          8,333
  贷：银行存款        10,614
```
4. 自动更新贷款的 `remaining_principal` 和 `repaid_months`
5. 全部还完后状态自动变为 `paid_off`

**提前还款：**

用户可在贷款详情页发起"提前还款"：
1. 输入提前还款金额
2. 选择还款账户
3. 生成还款分录（全部算本金，无利息）：
```
借：长期借款（本金）  50,000
  贷：银行存款        50,000
```
4. 更新 `remaining_principal`，若还清则状态变为 `paid_off`

## 4. 功能需求：预算管理 & 提醒

### 4.1 功能概述

为家庭月度费用支出设置预算上限，跟踪实际花费与预算的对比，超支时及时提醒。

**核心原则：预算只针对"费用类"科目（5xxx），购买资产（资产内部转换）不纳入费用预算。**

### 4.2 预算类型

| 类型 | 说明 |
|------|------|
| 总预算 | 所有费用科目的月度总额上限（`account_id = NULL`） |
| 分类预算 | 针对单个费用科目的月度上限（如：餐饮 3000/月） |

### 4.3 预算周期

- **月度预算**：当前版本仅支持月度周期
- 每月 1 日自动重置使用进度
- 预算金额可随时修改，修改后当月立即生效

### 4.4 预算计算

```
预算使用率 = 本月该科目实际费用 / 预算金额 × 100%

实际费用 = 该科目（或全部费用科目）在本月的借方合计
```

### 4.5 预算提醒规则

| 阈值 | 提醒方式 | 说明 |
|------|---------|------|
| 80% | 黄色警告 | "餐饮预算已使用 80%，剩余 ¥600" |
| 100% | 红色告警 | "餐饮预算已超支 ¥200" |
| 自定义 | 用户设定 | 用户可自定义提醒阈值（50%~100%） |

**提醒触发时机：**
- 每笔费用记账后，实时检查相关预算
- 如果该笔记账导致预算达到提醒阈值，弹出 Toast 提醒

### 4.6 与总览页联动
